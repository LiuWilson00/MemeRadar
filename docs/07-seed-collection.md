# Seed 蒐集指南（P0-3）

冷啟動的鐵律（docs/06 §3.2）：**按策略錨點配平，不是隨機收 300 張**。
庫太小時「檢索空手」會很頻繁，而配平決定了每種對話情境是否至少有圖可推。

## 1. 配額

| 項目 | 目標 | 說明 |
|------|------|------|
| 每策略錨點 | **≥ 8 張** | 14 個錨點 × 8 = 至少 112 張是底線 |
| 海綿寶寶 | **≥ 30 張** | Demo 指定主題，優先配足 |
| 甄嬛傳 | **≥ 30 張** | 同上（可與策略配額重疊計算） |
| 總量 | **150–300 張** | 寧缺勿濫，精選優於灌量 |

隨時檢查缺口：

```bash
python -m memeradar.ingestion.coverage
```

（匯入 + 標註後才會計入；報表逐錨點列出「✘ 缺 N 張」。）

## 2. 合規紅線（docs/02 §7 的立場，蒐集時同樣適用）

- **人工逐張存圖**，不用任何自動抓圖工具、不繞過網站的機器人阻擋。
- 用途限**個人研究 / 內部 Demo**，圖不對外重新發布。
- **記來源**：在 seed 根目錄放一份 `SOURCES.md`，一行一筆「檔名（或資料夾）→ 出處連結」。
  非圖片檔會被匯入腳本自動跳過，放在資料夾裡不礙事。
- 不收：含仇恨 / 歧視符號的圖（標註端會直接淘汰，別浪費配額）；
  名人政治類可少量收但檢索端預設排除；NSFW 靠標註端打標，蒐集時自行斟酌。

## 3. 資料夾組織

**子資料夾名會餵給標註模型當上下文提示**（`post_title`），所以用錨點 / 主題命名能直接提升標註品質：

```
seed/
  SOURCES.md          ← 來源紀錄（不會被匯入）
  滑跪求饒/
    對不起跪了.png
  安撫/
  嗆聲反擊/
  海綿寶寶/            ← 主題包：混合多種策略也沒關係，franchise 會被標出來
  甄嬛傳/
```

一張圖只放一個資料夾即可（sha256 去重，重複放也不會進兩次）。

## 5. 品質檢核（存圖前過一遍）

- 短邊 ≥ 200px（過小只警告仍會匯入，但檢索體驗差）
- 字要讀得清楚（OCR 是檢索文件的一部分）
- 繁中或無字圖優先；簡中可收但比例別過半
- 避開大浮水印、被裁切到失去梗點的圖

## 6. 進庫流程

```bash
# 1. 批次匯入（冪等，可重跑；會印每資料夾張數）
python -m memeradar.ingestion.seed_import <seed資料夾>

# 2. 批次標註（需 ANTHROPIC_API_KEY；已標註的不重複計費）
python -m memeradar.understanding.annotator

# 3. 向量化（需 local-embedding extras）
python -m memeradar.understanding.embedding

# 4. 看配平缺口，決定下一輪收什麼
python -m memeradar.ingestion.coverage
```

低信心 / 疑似非梗圖的標註會進 Console「複核」頁，蒐集告一段落後去掃一遍；
「報表」頁的 👎 歸因中「庫缺圖」類回饋，就是下一輪蒐集清單的直接輸入。

## 7. 最省力路線：Console 批次拖曳

不想碰終端機的話，收好圖後直接開 Console「**上傳**」頁，把圖**整批拖進去**即可——
服務會逐張跑「入庫 → 標註 → 向量化」，即時顯示每張狀態，重複自動略過。
拖之前可填一個整批共用的「主題提示」（例如「海綿寶寶」），會餵給標註模型當上下文。
上傳全程在瀏覽器操作，等同上面第 6 節前三步；只有第 4 步「看配平缺口」目前仍走
`python -m memeradar.ingestion.coverage`。零星單張補圖用「梗圖庫 → ＋上傳梗圖」也可以。
