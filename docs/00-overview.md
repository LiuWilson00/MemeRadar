# MemeRadar — 產品總覽

> AI 梗圖回應推薦系統：根據用戶的對話上下文，自動精準推薦適合回覆的梗圖。

## 1. 產品定位

MemeRadar 是一個從 0 到 1 的產品。使用者在聊天軟體（LINE / Messenger / Discord…）中遇到「這裡好想丟一張梗圖但找不到」的瞬間，把對話貼上（或截圖上傳）給 MemeRadar，系統即回傳 Top 3–5 張最適合當下情境的梗圖，並說明「為什麼推薦這張」。

**核心價值主張**：梗圖回覆的痛點不是「沒有梗圖」，而是「在對的時機找不到對的那張」。MemeRadar 把「理解對話 → 理解梗圖 → 精準配對」自動化。

## 2. 典型使用情境（User Stories）

| # | 情境 | 期望行為 |
|---|------|---------|
| U1 | 朋友在群組抱怨加班，我想回一張「無奈認命」的梗圖 | 貼上對話 → 推薦 3–5 張「無奈 / 認命 / 社畜」情境的梗圖，附推薦理由 |
| U2 | 對方在生氣，我想「滑跪」道歉 | 系統判斷對方情緒為憤怒 → 推薦「滑跪 / 求饒 / 裝可憐」類梗圖 |
| U3 | 我只想用海綿寶寶系列回覆 | 在過濾條件勾選「海綿寶寶」→ 結果僅限該主題 |
| U4 | 我懶得打字，直接截圖 LINE 對話上傳 | 系統 OCR + 解析氣泡左右方 → 還原對話結構後推薦 |
| U5 | 內部測試者覺得某張推得很爛 | 按 👎 並留備註 → 回饋落庫，作為日後調參 / 微調依據 |

## 4. 核心設計原則

1. **資料品質優先於數量**。500 張標註精準的梗圖，勝過 5 萬張雜訊。梗圖檢索是「命中率遊戲」——推薦 5 張裡只要 1 張爆準，體驗就成立；5 張都「有點像但不好笑」，體驗就崩潰。
2. **先打通端到端，再擴大資料**。Phase 0–2 用人工精選的 seed 資料集（150–300 張）把「標註 → 索引 → 匹配 → Demo」整條鏈路跑通並可評估，爬蟲自動化放在 Phase 3。避免一開始就陷在爬蟲與反爬對抗裡。
3. **每個推薦都要可解釋**。Console 上每張圖必須顯示「為何推薦」（命中的意圖、標籤、分數），這既是 Demo 說服力，也是開發者調校準確度的核心工具。
4. **回饋迴圈是一級公民**。👍/👎 與參數快照從第一天就落庫，累積成未來 rerank 調參與微調（fine-tuning）的資料資產。
5. **梗圖的靈魂在「使用情境」**。標註的核心欄位不是「圖裡有什麼」，而是「這張圖通常在什麼時候丟出來」（usage hint）。檢索匹配的主軸是 *對話情境 ↔ 使用情境*，不是 *對話文字 ↔ 圖片描述*。

## 5. 名詞定義

| 名詞 | 定義 |
|------|------|
| 梗圖（Meme） | 帶有網路文化語境、可用於對話回應的圖片。同一模板配上不同文字視為**不同**梗圖 |
| 模板（Template） | 梗圖的底圖來源（如「派大星攤手」），一個模板可衍生多張梗圖 |
| 使用情境（Usage Hint） | 一句話描述「這張圖通常什麼時候用」，例如「當對方提出離譜要求時表達傻眼」 |
| 回應策略（Response Strategy） | 對同一段對話可採取的回覆姿態：安撫、嘲諷、自嘲、滑跪、裝傻、慶祝… |
| 梗圖包（Meme Pack） | 依主題 / 作品（franchise）分組的梗圖集合，如「海綿寶寶」「甄嬛傳」 |
| 檢索文件（Retrieval Document） | 由 OCR 文字 + 圖片描述 + 使用情境 + 標籤組裝成的一段文字，是 embedding 的輸入 |
| Punchline | 對話中觸發回應的關鍵爆點句，意圖分析的錨點 |
| Golden Set | 人工標註的「對話 → 適合梗圖」評測集，用於離線評估 Recall@K |

## 6. MVP 範圍界定

**In scope（Phase 0–4）**

- 靜態圖片（JPG / PNG / WebP）
- 繁體中文為主的對話與梗圖（可容忍中英夾雜）
- 內部 Demo / 調適用 Web Console（無帳號系統、不對外）
- 資料源：Reddit、Dcard 等 2–3 個高純度來源 + 人工 seed

**Out of scope（明確不做，未來再議）**

- GIF / 影片梗（Phase 1 直接排除或僅取首幀）
- 對外公開產品、用戶帳號、行動 App
- 梗圖生成（本系統是「檢索推薦」，不是「生圖」）
- 完整多語系支援
- 即時聊天軟體外掛 / Bot 整合

## 7. 成功指標（Demo 階段）

| 指標 | 目標 | 量測方式 |
|------|------|---------|
| Recall@5（golden set） | ≥ 60% | 離線評測：正解梗圖出現在 Top 5 的比例 |
| 測試者滿意率（👍 / 總回饋） | ≥ 50% | Console 回饋事件統計 |
| 端到端延遲（文字輸入） | ≤ 15 秒（2026-07-11 依實測修訂，實測 ~13s；8 秒為後續優化目標） | API 計時 |
| 端到端延遲（截圖輸入） | ≤ 22 秒（同上修訂，含截圖解析） | API 計時（含截圖解析） |
| 梗圖庫規模（Phase 3 後） | ≥ 3,000 張（去重後） | 資料庫統計 |
